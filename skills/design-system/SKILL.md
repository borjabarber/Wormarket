# Objetivo

Mantener una identidad visual coherente para Wormarket sin sacrificar claridad de marketplace.

# Cuando utilizarla

- Tokens visuales.
- Componentes reutilizables.
- Patrones de layout.
- Estados de UI.
- Cards de anuncios.
- Formularios y navegacion.

# Cuando no utilizarla

- Reglas de dominio.
- Persistencia.
- Despliegue.

# Procedimiento

1. Lee `VISUAL_DIRECTION.md` y `DESIGN_SYSTEM.md`.
2. Parte de flujos reales de usuario.
3. Usa componentes familiares.
4. Evita decoracion que impida escanear productos.
5. Mantiene cards con radio moderado.
6. Verifica contraste, foco y responsive.
7. Documenta nuevos componentes en `COMPONENTS.md`.

# Comprobaciones

- Coherencia con tokens.
- Responsive.
- Accesibilidad AA.
- Estados de carga, vacio y error.

# Resultado esperado

UI consistente, legible y distintiva para Wormarket.
